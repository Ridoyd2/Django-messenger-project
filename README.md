# Django Messenger with AI Bot Responses

This is a messaging web application built with Django that features AI responses for offline users. When a user is offline and has the AI bot enabled, the system analyzes past conversations and generates context-aware AI responses.

## Features

- User authentication (login, logout, signup)
- Real-time message updates
- Online/offline status tracking
- Unread message counting
- AI-powered responses using Hugging Face API
- Togglable AI bot for each user
- Smart fallback responses when AI API is unavailable
- Clean, responsive UI

## Technology Stack

- Django 5.1.7
- jQuery for AJAX calls
- Bootstrap 5 for UI
- Hugging Face API for AI responses

## Installation

1. Clone the repository
2. Install dependencies:
```
pip install -r requirements.txt
```

## Configuration

The application uses Hugging Face API for generating AI responses. The API key is configured in the settings file. You can replace it with your own API key by setting the `HUGGINGFACE_API_KEY` environment variable.

## Running the Application

1. Apply migrations:
```
python manage.py migrate
```

2. Create a demo user (optional):
```
python create_demo_user.py
```

3. Run the development server:
```
python manage.py runserver
```

4. Access the application at http://127.0.0.1:8000/

## How the AI Response Works

When a user sends a message to an offline user:
1. The system checks if the recipient is offline AND has the AI bot enabled
2. If both conditions are met, it retrieves the last 10 messages between the two users
3. It attempts to create a context-rich prompt and send it to the Hugging Face API
4. If successful, the API generates a response that mimics the offline user's conversational style
5. If the API fails or times out, the system falls back to smart contextual responses based on message content
6. The response is saved as a bot message and sent back to the sender

## AI Bot Toggle

Each user can toggle their AI bot on or off from the home page. When enabled:
- The AI will respond on your behalf when you're offline
- It will analyze your conversation history to respond in a similar style
- Other users will see an "AI Response" indicator on messages generated by the bot

## User Status Management

The application automatically tracks user status:
- When a user logs in, they are marked as "Online"
- When a user logs out, they are marked as "Offline"
- Staff users can view and manage all user statuses on the Active Sessions page
- The Active Sessions page allows forcing users offline if needed
